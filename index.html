Software Design and Engineering – 3D Office Scene

A graphical program created with OpenGL for the CS 330: Computational Graphics and Visualization course is called the 3D Office Scene made in 2024. The original goal of the project was to illustrate fundamental ideas in computer graphics, such as scene design, lighting, texturing, camera movement, and 3D transformations. The artifact allows the user to interactively traverse a virtual office setting made up of several things placed in a three-dimensional area. In order to better represent professional software engineering techniques, this project which was finished earlier in class now was reexamined in CS 499.

I choose to include this artifact in my ePortfolio because it effectively demonstrates my skills in applied mathematics, modular programming, and software design inside a sophisticated visual system. Through the separation of rendering functionality from application control, organized scene management, and reusable object classes, the project demonstrates object-oriented design. Particular elements like shader programs, lighting computations, camera controls, and texture mapping demonstrate my comprehension of both higher-level architectural choices and low-level graphics pipelines. By increasing lighting realism, reorganizing rendering components into more modular structures, and restructuring code for clarity and maintainability, the improvement phase enhanced the artifact. Refactoring, performance awareness, debugging graphical systems, and applying design concepts to an existing codebase are all demonstrated by these improvements.

Enhancing this artifact was a valuable learning experience that reinforced the importance of clean architecture and iterative improvement in software development. One of the main challenges was modifying and extending existing graphics code without breaking rendering behavior, which required careful testing and incremental changes. I learned how small changes in shaders, transformations, or lighting calculations can have significant visual effects, emphasizing the need for precision and documentation. My improvement technique was shaped by teacher instruction and feedback from the initial project, especially in terms of increasing the organization and clarity of the code. Through this process, I partially met course outcomes related to cross-platform optimization and documentation depth, which could be further improved with more time and testing. However, I fully met course outcomes related to software design, problem-solving, and the application of computing principles.
